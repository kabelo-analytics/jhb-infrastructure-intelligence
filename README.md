# JHB Municipal Infrastructure Intelligence
## Potholes + Water Leakages

A portfolio analytics project that simulates how a City of Johannesburg infrastructure team could monitor service delivery for potholes and water leakages using a synthetic relational dataset and operational KPIs.

This repo is Python-based and currently covers data generation, cleaning, feature building, notebook analysis, and lightweight HTML reporting. A Power BI layer is still planned separately.

---

## Why this matters (Johannesburg context)
Johannesburg faces recurring infrastructure pressure: ageing assets, burst pipes, water leaks, and road deterioration. This project builds a reporting structure that supports:
- SLA monitoring
- backlog visibility
- ward-level incident monitoring
- contractor comparison
- operational KPI storytelling

---

## Project goals
1. Build a synthetic dataset for pothole and water leak incidents.
2. Produce core operational metrics:
   - time-to-respond, time-to-repair, SLA breach rates
   - backlog size and trend
   - ward-level incident volume
3. Create notebook outputs and exported figures suitable for portfolio review.
4. Keep the project extensible for a later BI dashboard layer.

## Quickstart
### 1) Create environment
```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

### 2) Generate synthetic dataset (seeded)
```bash
python -m src.generate.generator --seed 753 --rows 200000
```

### 3) Clean + validate
```bash
python -m src.clean.cleaners
python -m src.utils.validation
```

### 4) Build analysis features
```bash
python -m src.features.build_features
```

### 5) Run analysis notebook
Open `notebooks/03_eda_kpis.ipynb` and execute.

## Current supportable KPI outputs
Using the seeded synthetic dataset generated by the shipped code:
- Total incidents analysed: 200,000
- Wards represented: 135
- Contractors represented: 6
- Median time-to-first-response: 24.0 hours
- Median time-to-repair: 3.55 days
- SLA breach rate: 16.4%

These outputs are demonstrated in `reports/report.md` and the notebook workflow.
